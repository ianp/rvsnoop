Release Notes

Quick Start

You can get started by running the scripts in the bin directory, there is one for Windows and one for Unix.

More detailed information can be found in the local help files.

Summary of Changes

Some general changes in terminology have taken place in the documentation. The tree view on the left of the window is now known as the ‘Subject Explorer’ since that is what it actually shows, and the main table view is the ‘Message Ledger’ since again, that is what it shows. The old ‘Category’ names were a holdover from Log Factor 5. Some areas in the UI and documentation may still use the old names, if you spot any please mark it as a bug on SourceForge.

In a similar vein, there has been some class renaming. I feel that this is OK at this stage as RvSn00p has never had a public API before. The idea is, in broad terms, to introduce some scriptablility around the 2.0 timeframe, and there needs to be a clean public API before then. For an example of what has changed: rvsn00p.LogRecord is now rvsn00p.Record. There will be more changes like this in the upcoming point releases, but it will cease once 2.0 is announced.

The standard cut/copy/paste menu actions have been added. Also, it is now possible to delete records from the ledger and to prune empty subjects from the subject explorer.

Future Plans

Please suggest further enhancements via the SourceForge tracker.

Main plans for the next version are the addition of a view of the current listeners, along with listener details and listener specific actions (e.g. pause a single listener). Also, the export to HTML facility will be altered to use XHTML 1.0 Strict. Finally, the ability to resend message may be added (this may slip to the following version however).

There is also ongoing work to clean-up and refactoring the code base with a view to having a public API as part of the 2.0 release (this will allow scripting).
