Release Notes

RvSn00p hasn’t changed for quite a while so there are a number of new features (and hopefully no new bugs!) in this release. This document summarizes the new features.

Quick Start

You can get started by running the scripts in the bin directory, there is one for Windows and one for Unix.

More detailed information can be found in the local help files.

Summary of Changes

It is now possible to save your configuration and re-open it at a later date. This is done using the XOM 1.1 library and so handles encoding odd-characters well, the older, command line based, means of starting up preconfigured listeners still works, but is much more limited. For the next next release the old mechanism will probably be removed.

The details panel now uses a graphical tree view to display the message. At the moment this just displays the raw field structure of the message, it does not attempt to display Active Enterprise objects. One additional feature however, is the ability to parse and display XML in the same tree as the message, this is currently only enabled if the field uses the TIBCO XML field type, it will not work if you have XML data stored as a string.

One point to note: the Raccoon marshaller has been removed, it referenced an extremely old version of the Raccoon framework and would not compile against the current code.

Future Plans

Please suggest further enhancements via the SourceForge tracker.

The next release after this will probably focus on code clean-up and refactoring rather than adding new features. Also, the current code can now be run through the internal static analysis tools in Eclipse with no warnings, the next step is to run some more code analysis tools and then, once any/all obvious problems have been fixed, write some unit tests.
