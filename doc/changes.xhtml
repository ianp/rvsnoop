Change Log

Request ID Numbers refer to bug requests or feature requests on the SourceForge page.

Changes up to version 1.3.0

  • The configuration is saved and opened with File → Open File → Save.
  • Listeners are stored in the configuration file.
  • A new tree view has been implemented for the message display, this is displayed along with the text view in a tabbed panel. The old style text-only view can be used via the -textonly switch. The tabbed view can be suppressed, displaying only the new style tree, via the -treeonly switch.
  • Finished adding a "recent listeners" facility to the file menu. These are serialized to an XML file when the application shuts down.
  • Improved the start scripts.
  • Updated ASF license to use version 2.0.
  • Updated copyright notices for new license.
  • Fixed bug in RvParameters object whereby the hash code could change over the objects life-cycle, thus preventing it from being added to collections.
  • Updated the configuration manager to use XOM to read and write the XML based configuration files, this ensures that they are always well formed XML. A DTD for the configuration file will be added in a later version.
  • A new build system.
  • Cleaned up lots of code so that the Eclipse code checker no longer generates any warnings.
  • Removed the raccoon code as it does not compile against the current version of Raccoon.
  • Corrected some minor spelling mistakes.

Changes up to version 1.2.6

  • Java 1.4.1 or later is required.
  • Added Raccoon message serializer, this will be used if the Raccoon classes are on the classpath.
  • Added Reuters RvTest serializer, this will be used if the RvTest classes are on the classpath.
  • RvSn00p now supports the Raccoon, RvScript and RvTest serializers.
  • Other minor changes.

Changes up to version 1.2.4

  • Added operation Edit → Filter by Selected Subject with accelerator key ctrl-y.
  • Added operation Edit → Filter by Subject with accelerator key ctrl-shift-y.

Changes up to version 1.2.3

  • Configure → Save will now save: window position, table column width, splitter pane positions, date format, font settings

Changes up to version 1.2.1

  • Minimized number of objects sent to garbage collection (more work on this to follow). I used CompuWare DevPartner for this. Remember to use the Configure → Set Max Number of Records to limit the number of records contained in memory.
  • Batched creation of new log information objects, thus increasing performance under high load.
  • Changed HTML output format to include intendation of RvScript XML.

Changes up to version 1.1.11

  • Tracking ID column in table. If a MTrackingInfo class is found in the message it's tracking id is put into this field.
  • Operation Edit → Filter by Tracking ID.
  • Operation Edit → Remove Tracking ID Filter.
  • Added ability to supply multiple subjects when setting up listeners, i.e. the command line argument "tcp:7500|7500||subja,subjb" will listen for subja and subjb on the daemon tcp:7500 on servce 7500. Adding another subject subjc is done by just using the "Add listener".

Changes up to version 1.1.6

  • New help options for licence information.
  • Changed implementation of RvMsg string serialization to allow later implemetation of editing and resending messages.

  • RvSn00p now uses the first found of:

    1. TIBCO RvScript tibrvXmlConvert
    2. MTree.toString
    3. TibRvMsg.toString

    This means that if the rvscript.jar is in the classpath tibrvXmlConvert will be used as string serializer. If it is not found the serializer will be downgraded to the TIBCO SDK AE ‘MTree’ format and if that is not not found the standard RV API TibRvMsg will be used.
